This is the complete, cleaned-up implementation for the **Staged Services** feature. It includes the Prisma schema, the DTOs, the Service logic with Redis caching, and the Controller.

### 1. Prisma Schema
Add this model to your `schema.prisma` file. This acts as the buffer for imported data before it is finalized.

```prisma
model StagedService {
  id              String   @id @default(uuid())
  salonId         String
  externalId      String?  // Original ID from CSV (e.g., #001)
  
  name            String
  description     String?
  duration        Int      @default(0)
  price           Float    @default(0)
  vat             Float    @default(0)
  category        String?
  postBreakMin    Int      @default(0)
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@index([salonId])
}
```

### 3. Service Logic
This service handles data normalization (converting strings to numbers) and uses a "hard delete" strategy for cleanup as per your preference.

```typescript
/* eslint-disable @typescript-eslint/no-unsafe-return */
/* eslint-disable @typescript-eslint/no-unsafe-assignment */
/* eslint-disable @typescript-eslint/no-unsafe-member-access */
/* eslint-disable @typescript-eslint/no-unsafe-call */
import { Injectable, BadRequestException } from '@nestjs/common';
import { PrismaService } from '../../../core/prisma/prisma.service';
import { RedisService } from '../../../core/redis/redis.service';
import { SERVICE_CACHE_PREFIX } from '../../../core/redis/redis.constants';

@Injectable()
export class ServiceImportService {
  private readonly CACHE_PREFIX = 'staged_services_import:v1';

  constructor(
    private readonly prisma: PrismaService,
    private readonly redis: RedisService,
  ) {}

  async stageData(salonId: string, items: any[]) {
    // Map raw data and normalize numeric types
    await this.prisma.stagedService.createMany({
      data: items.map((item) => ({
        salonId,
        externalId: item.id?.toString(),
        name: item.name || 'Unnamed Service',
        description: item.description,
        duration: parseInt(item.default_duration) || 0,
        price: parseFloat(item.price) || 0,
        vat: parseFloat(item.vat) || 0,
        category: item.category,
        postBreakMin: parseInt(item.post_break_min) || 0,
      })),
    });
    await this.redis.flushByPrefix(`${this.CACHE_PREFIX}:${salonId}`);
  }

  async getStagedData(salonId: string, page = 1, limit = 10) {
    const cacheKey = `${this.CACHE_PREFIX}:${salonId}:${page}:${limit}`;
    const cached = await this.redis.get(cacheKey);
    if (cached) return JSON.parse(cached as string);

    const skip = (page - 1) * limit;
    const [total, data] = await Promise.all([
      this.prisma.stagedService.count({ where: { salonId } }),
      this.prisma.stagedService.findMany({
        where: { salonId },
        skip,
        take: limit,
        orderBy: { createdAt: 'desc' },
      }),
    ]);

    const result = {
      data,
      meta: { total, page, lastPage: Math.ceil(total / limit) },
    };
    
    await this.redis.set(cacheKey, JSON.stringify(result), 60);
    return result;
  }

  async approveBulk(salonId: string, ids: string[]) {
    const items = await this.prisma.stagedService.findMany({
      where: { id: { in: ids }, salonId },
    });

    if (items.length === 0) {
      throw new BadRequestException('No services found to approve');
    }

    return await this.prisma.$transaction(async (tx) => {
      for (const item of items) {
        await tx.service.create({
          data: {
            salonId: salonId,
            serviceName: item.name,
            description: item.description,
            duration: item.duration,
            price: item.price,
            vat: item.vat,
            categories: item.category ? [item.category] : [],
            postBreakMin: item.postBreakMin,
          },
        });
      }

      // Permanent hard delete after approval
      await tx.stagedService.deleteMany({
        where: { id: { in: ids }, salonId },
      });

      // Clear both import and live service caches
      await this.redis.flushByPrefix(`${this.CACHE_PREFIX}:${salonId}`);
      await this.redis.flushByPrefix(SERVICE_CACHE_PREFIX);

      return { success: true, count: items.length };
    });
  }

  async deleteStaged(salonId: string, ids: string[]) {
    const result = await this.prisma.stagedService.deleteMany({
      where: { id: { in: ids }, salonId },
    });
    await this.redis.flushByPrefix(`${this.CACHE_PREFIX}:${salonId}`);
    return result;
  }
}
```

### 4. Controller
Mapped to the tenancy architecture for salon-specific access.

```typescript
/* eslint-disable @typescript-eslint/no-unsafe-argument */
import {
  Controller,
  Post,
  Get,
  Delete,
  Body,
  Query,
  Req,
  UseGuards,
} from '@nestjs/common';
import { ServiceImportService } from './import-service.service';
import { JwtAuthGuard } from '../../../core/guards/jwt-auth.guard';
import {
  StageServiceImportDto,
  BulkActionDto,
  ImportQueryDto,
} from './dto/import-service.dto';

@Controller('tenancy/services/import')
@UseGuards(JwtAuthGuard)
export class ServiceImportController {
  constructor(private readonly service: ServiceImportService) {}

  @Post('stage')
  async upload(@Req() req: any, @Body() dto: StageServiceImportDto) {
    return this.service.stageData(req.user.salonId, dto.items);
  }

  @Get('staged-list')
  async getList(@Req() req: any, @Query() query: ImportQueryDto) {
    return this.service.getStagedData(
      req.user.salonId,
      Number(query.page) || 1,
      Number(query.limit) || 10,
    );
  }

  @Post('approve-mass')
  async approve(@Req() req: any, @Body() dto: BulkActionDto) {
    return this.service.approveBulk(req.user.salonId, dto.ids);
  }

  @Delete('reject-mass')
  async reject(@Req() req: any, @Body() dto: BulkActionDto) {
    return this.service.deleteStaged(req.user.salonId, dto.ids);
  }
}
```

### Key Technical Details:
* **Redis Caching**: The `getStagedData` method implements caching with a 60-second TTL to ensure high performance for paginated table views.
* **Transaction Safety**: `approveBulk` uses a `$transaction` to ensure that either all selected services are moved to the live table, or none are, preventing partial data corruption.
* **Normalization**: Numeric inputs like `price` and `duration` are parsed manually to handle strings coming from CSV uploads.



Testing these APIs in Postman requires a specific sequence because of the "Staging" architecture. You first upload the raw data to a buffer, then verify it, and finally move it to the live services table.

Ensure you have your **JWT Token** ready from your login endpoint, as these routes are protected by the `JwtAuthGuard`.

## 1. Environment Setup
In Postman, create an environment or use global variables for:
* `base_url`: `http://localhost:3000` (or your server URL)
* `token`: Your actual JWT access token

In the **Authorization** tab of your Postman folder, set the Type to **Bearer Token** and use the `{{token}}` variable.

## 2. The Testing Workflow

### Step 1: Stage the Data (Upload)
This mimics the "Import" action where you send raw JSON/CSV data.

* **Method:** `POST`
* **URL:** `{{base_url}}/tenancy/services/import/stage`
* **Body (raw JSON):**
```json
{
  "items": [
    {
      "id": "#101",
      "name": "Luxury Haircut",
      "description": "Includes wash and style",
      "default_duration": "45",
      "price": "250",
      "vat": "50",
      "category": "Hair",
      "post_break_min": "15"
    },
    {
      "id": "#102",
      "name": "Beard Grooming",
      "description": "Trim and oil",
      "default_duration": "30",
      "price": "100",
      "vat": "20",
      "category": "Grooming",
      "post_break_min": "5"
    }
  ]
}
```

### Step 2: View Staged List (Review)
This verifies that your logic correctly saved the items and handled the Redis caching.

* **Method:** `GET`
* **URL:** `{{base_url}}/tenancy/services/import/staged-list?page=1&limit=10`
* **Expected Behavior:** You should see the items with generated UUIDs from the `StagedService` model. **Copy the `id` values** from the response for the next step.

### Step 3: Approve in Bulk (Finalize)
This moves the items to your live `Service` table and performs a **hard delete** on the staging rows.

* **Method:** `POST`
* **URL:** `{{base_url}}/tenancy/services/import/approve-mass`
* **Body (raw JSON):**
```json
{
  "ids": ["paste-uuid-1-here", "paste-uuid-2-here"]
}
```

### Step 4: Verify Live Data
Check your original Services Search API to ensure the new services appear.

* **Method:** `GET`
* **URL:** `{{base_url}}/salons/:salonId/services/search`
* **Expected Behavior:** The "Luxury Haircut" should now be visible in your main service list.

## 3. Postman Pro-Tips for your Stack

* **Pagination Test:** Run Step 2 with `?limit=1`. Check the `meta` object in the response to ensure `lastPage` and `total` are calculating correctly based on your `Prisma.count` logic.
* **Cache Invalidation Test:** Perform Step 2, then Step 3. If you call Step 2 again, the items you approved should immediately disappear because the code calls `redis.flushByPrefix` after approval.
* **Error Handling:** Try sending Step 3 with a random UUID that doesn't exist. You should receive a **400 Bad Request** with "No services found to approve" as defined in your service logic.
